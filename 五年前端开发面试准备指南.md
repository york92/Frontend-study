下面按**“5年前端开发”**的面试标准来准备。你要先明确一点：**5年经验的面试，已经不是单纯问你会不会写页面，而是看你能不能独立负责模块、解决复杂问题、做技术取舍、推动项目落地。**

# 一、5年前端面试官到底在看什么？

5年前端一般被定位为：**中高级前端 / 高级前端候选人**。

面试官重点看这几件事：

1. **基础是否扎实**
   JS、TS、CSS、浏览器、网络、异步、工程化不能只会用，要能解释原理。

2. **框架是否深入**
   React / Vue 不只是会写组件，而是要理解响应式、渲染机制、状态管理、性能优化、组件设计。

3. **项目是否有深度**
   不能只说“我负责某某页面”，而要说清楚：业务背景、技术难点、方案设计、性能指标、踩坑、收益。

4. **工程化能力**
   Vite、Webpack、Monorepo、CI/CD、代码规范、构建优化、发布流程、灰度、埋点、错误监控。

5. **性能、安全、稳定性意识**
   例如首屏优化、长列表优化、接口并发、缓存策略、XSS/CSRF、防重复提交、异常兜底。

6. **架构设计能力**
   给你一个后台管理系统、低代码平台、组件库、权限系统、埋点系统、文件上传系统，你能不能设计出来。

7. **沟通协作能力**
   5年经验会被默认需要和产品、后端、测试、设计、新人协作，而不是只等需求写代码。

# 二、面试流程一般分几轮？

## 1. 简历筛选阶段

简历最重要的不是写“熟悉 Vue、React、Webpack”，而是要体现：

**你做过什么复杂事情，解决了什么问题，带来了什么结果。**

例如不要这样写：

> 负责后台管理系统开发，使用 Vue3 + Element Plus 完成页面开发。

应该这样写：

> 负责企业级后台管理系统核心模块开发，基于 Vue3 + TypeScript 封装动态表单、权限路由、通用表格和文件上传组件，减少重复页面开发约 40%，并通过路由懒加载、组件按需加载、接口缓存等方式优化首屏加载体验。

5年经验的简历一定要有这些内容：

* 项目背景
* 技术栈
* 个人职责
* 技术难点
* 解决方案
* 量化结果
* 线上问题处理经验
* 性能优化经验
* 组件封装 / 工程化 / 架构设计经验

---

## 2. 一面：基础 + 手写 + 框架

一面通常会问：

* JS 基础
* CSS 布局
* 浏览器原理
* HTTP
* Promise / async / await
* 原型链 / 闭包 / 事件循环
* React / Vue 原理
* 简单算法
* 手写代码

这轮考察的是：**你是不是“真会前端”。**

---

## 3. 二面：项目深挖 + 工程化 + 架构

二面会重点问：

* 你项目里最难的问题是什么？
* 为什么这样设计？
* 有没有其他方案？
* 如果数据量扩大 10 倍怎么办？
* 如果接口很慢怎么办？
* 如果用户权限很复杂怎么办？
* 如果多人协作，组件如何拆？
* 线上出问题你怎么排查？
* 你做过哪些性能优化？
* 你怎么保证代码质量？

这轮考察的是：**你是不是能独立负责复杂模块。**

---

## 4. 三面 / 主管面：综合能力 + 业务理解

这一轮更偏：

* 你为什么离职？
* 你怎么看前端发展？
* 你怎么推动技术方案落地？
* 你和后端/产品冲突怎么处理？
* 你有没有带人经验？
* 你能不能从业务角度考虑问题？
* 你未来想往技术专家还是管理发展？

这轮考察的是：**你是不是能长期稳定地承担团队价值。**

> 为什么 transform 比 top/left 动画性能好？

你要能回答：

# 十三、5年前端最容易被刷的原因

## 1. 项目讲得太浅

只说：

> 我负责页面开发，调接口，写组件。

这像 1—2 年经验。

5年经验要说：

> 我负责模块设计、组件抽象、权限接入、接口封装、性能优化、上线排查、后续维护。

---

## 2. 八股背了但不会结合项目

例如问性能优化，你背了一堆：

* 懒加载
* 缓存
* CDN
* 压缩
* 防抖节流

但面试官继续问：

> 你项目里具体优化了哪个页面？优化前后指标是多少？

答不上来就很危险。

---

## 3. 框架只会用，不懂原理

5年经验如果 Vue 只会写 `ref`、`reactive`，React 只会写 `useState`、`useEffect`，会显得深度不足。

你至少要能讲清楚：

* 响应式如何追踪依赖
* 为什么组件会重新渲染
* key 的作用
* diff 的基本思路
* 状态更新为什么不是立即同步到 DOM
* 如何减少无效渲染

---

## 4. 没有工程化意识

如果你说：

> 项目都是别人搭好的，我主要写业务。

对于5年经验会比较减分。

你要至少能讲：

* 项目目录为什么这样设计
* 请求层怎么封装
* 权限怎么接
* 环境变量怎么管理
* 打包体积怎么看
* ESLint 怎么接
* Git 提交规范怎么做
* 出问题怎么回滚

---

## 5. 没有业务理解

5年前端要能从业务角度思考。

比如不要只说：

> 这个功能我实现了。

要说：

> 这个功能是为了解决运营配置效率低的问题，所以我把固定页面改成 schema 驱动，后续新增配置项不用重新发版。

这就是高级感。

# 十四、建议你重点准备的“王牌项目表达”

你可以准备一个类似这样的项目作为主打：

## 项目：企业级后台管理系统 / SaaS 平台

重点包装方向：

1. 权限系统
   动态路由、菜单权限、按钮权限、接口权限、角色配置。

2. 动态表单
   schema 配置、字段联动、异步数据源、校验规则、回显转换。

3. 通用表格
   查询、分页、排序、操作列、权限按钮、导出、列配置。

4. 请求封装
   Token 刷新、错误码处理、取消重复请求、接口重试、loading 合并。

5. 性能优化
   路由懒加载、组件懒加载、缓存、虚拟列表、打包分析。

6. 工程化
   Vite、TypeScript、ESLint、Prettier、环境变量、自动部署。

这样一个项目可以覆盖 70% 的中高级前端面试问题。

# 十五、面试回答万能结构

回答复杂问题时，用这个结构：

```text
1. 先说结论
2. 再说背景
3. 解释为什么这么做
4. 说明具体实现
5. 补充边界情况
6. 说结果和复盘
```

例如：

> 你们权限系统怎么做的？

可以回答：

```text
我们权限分成菜单权限、路由权限、按钮权限和接口权限。

前端登录后获取用户信息和权限码，根据后端返回的菜单树生成动态路由，同时在页面中通过权限指令或权限函数控制按钮显示。

但是我不会把前端权限当成真正的安全边界，因为前端代码可以被篡改，所以接口权限必须由后端再次校验。

为了避免刷新丢失动态路由，我们会在初始化应用时重新拉取用户权限并生成路由，同时处理 401、403、token 过期和无权限页面跳转。
```

这个回答就比“用路由守卫判断权限”高级很多。

# 十七、一句话总结

5年前端面试的核心不是“我会写页面”，而是：

> **我能独立负责复杂模块，能做技术方案，能解决线上问题，能优化性能，能把业务需求稳定落地。**

你准备时不要只背八股，要把每个知识点都和自己的项目绑定起来。面试官真正想听的是：**你用这些技术解决过什么真实问题。**
